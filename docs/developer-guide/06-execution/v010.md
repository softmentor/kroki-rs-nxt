### 2026-03-04 (Standard Kroki API Endpoints)

- Implemented wire-compatible standard Kroki endpoints:
  - `POST /{type}/{format}` — raw text body (`Content-Type: text/plain`)
  - `POST /` — JSON body (`{"diagram_type", "output_format", "diagram_source"}`)
  - `GET /{type}/{format}/{encoded}` — base64+deflate encoded source in URL
- Retained legacy `POST /render` endpoint for backward compatibility.
- All endpoints share the same render pipeline through transport adapter.

### 2026-03-04 (CLI Encode/Decode/Version Commands)

- Implemented `kroki encode` — text to deflate + base64 (URL-safe, no padding).
- Implemented `kroki decode` — base64 + deflate to plain text, with fallback across URL-safe/standard variants.
- Implemented `kroki version` — prints crate version.
- All commands support stdin piping.
- Added encode/decode roundtrip test.

### 2026-03-04 (File Extension Auto-Detection)

- When `--diagram-type` is omitted, type is inferred from `--input` file extension.

### 2026-03-04 (RFC 7807 Problem Details Error Responses)

- Error type URIs follow `https://kroki.io/errors/{error_type}` convention.

- Added 2 verification tests checking content-type and JSON structure.
